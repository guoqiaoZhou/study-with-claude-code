# swcc · go — 开始复习

发起一次复习会话。**这是一次对话内的复习**:本技能加载上下文、围绕一个节点逐层考核;复习结束后用 `/swcc-stop` 归档。

> 开始前先读数据契约:`${CLAUDE_PLUGIN_ROOT}/skills/_shared/data-contract.md`。本技能**只读不写**——所有进度/掌握度的落盘由 `/swcc-stop` 完成。

参数:`$ARGUMENTS` —— 可选 topic（默认 config 的 activeTopic）、可选 mode（`concept`/`scenario`/`mixed`，默认 `mixed`）。

---

## 核心原则

1. **深挖，不是过一遍。** 每个概念都要问到「为什么 / 取舍 / 边界」，而不是答对结论就放过。会背 ≠ 掌握。
2. **覆盖该节点的关键概念。** 围绕一份核心概念清单逐个考核，别随机问一两题就散。
3. **一次只问一题。** 等用户答完、点评，再问下一题。
4. **挂了书就用书。** 出题与点评以原文为据，不泛泛而谈;没书则靠自身理解，但不假称来源。
5. **为 stop 铺路。** 收尾时输出结构化小结，让 stop 的评估子智能体能客观打分。

---

## 执行流程

| 阶段 | 名称 | 目的 |
|---|---|---|
| 1 | 加载专题 | 解析参数、读数据 |
| 2 | 定位节点 | 按优先级选当前该复习的节点 |
| 3 | 建覆盖清单 | 取该节点核心概念作为考核 checklist |
| 4 | 逐层考核 | 苏格拉底式追问，一次一题 |
| 5 | 收尾交接 | 输出结构化小结，提示 stop |

---

### 阶段 1：加载专题

- topic 缺省 → 读 `$HOME/.study-with-cc/config.json` 的 `activeTopic`（config 不存在或缺 activeTopic 时，按数据契约第九节「兜底」扫 `topics/` 自愈，不要直接报错）。
- 该专题不存在 → 提示先 `/swcc-plan <topic>`，停止。
- 读 `progress.json`、`knowledge-tree.md`、`knowledge-system.md`、`references.json`。

### 阶段 2：定位当前复习节点

按优先级:

| 优先级 | 选谁 |
|---|---|
| 1 | `weakPoints` 中 `nextReview` ≤ 今天（`date +%F`）的薄弱点——优先复习到期薄弱点 |
| 2 | 否则 `currentNode` 指向的节点（若为 `in_progress`） |
| 3 | 否则下一个 `not_started` 的叶子节点 |

### 阶段 3：建该节点的核心概念覆盖清单

从 knowledge-system.md 取**当前节点的「核心概念」清单**，作为本次会话的内部 checklist。本次围绕这些概念逐个考核，确保覆盖关键点（这也是 stop「覆盖度」评分的依据）。挂了书则按 data-contract 第八节读与本节点相关的章节页备用。

宣布定位信息:
```
🎯 当前复习：<topic> / <节点路径>
📅 上次复习：<lastReviewed 或「首次」>
🔄 模式：<mode>　核心概念待考核：<x> 个
```

### 阶段 4：逐层考核（苏格拉底式，一次一题）

围绕覆盖清单逐个概念考核。**每个概念不是问一题就过**，按用户回答质量动态调难度:

| 用户回答 | 你的下一步 |
|---|---|
| 答得准且深（讲清原理/取舍/边界） | 认可 → 升一档:追问边界/反例/与替代方案对比，或换下一个概念 |
| 答得对但浅（只背了结论） | 追问「为什么会这样」「换个场景还成立吗」，逼出原理 |
| 卡壳 / 答错 | 先给一个小提示再追;仍卡 → 简要补讲原理，**心里标记为本次薄弱点** |

- 题目结合 mode:`concept` 偏原理推导、`scenario` 给真实工程场景（秒杀/排行榜/故障排查类）、`mixed` 交替。
- 每题:提问 → 等答 → **点评**（指出对/错/缺漏，必要时补讲原理与取舍）→ 下一题。
- 围绕当前节点充分考核（尽量覆盖清单里的概念），不要一次跳太多节点。

> 提醒（原则 1）:用户答「对」时别急着夸完就走，再问一层「为什么」往往就见真章。

### 阶段 5：收尾交接

当节点关键概念已充分考核，或用户表示告一段落时，输出**结构化小结**（这是喂给 `/swcc-stop` 评估子智能体的关键输入）:

```
📝 本次复习小结（供 stop 评估用）
- 节点：<topic> / <节点路径>　模式：<mode>
- 覆盖的核心概念：<逐个列，标注 ✅充分 / ⚠️粗略 / ❌未答出>
- 答得好的点：<…>
- 卡壳/答浅的点（候选薄弱点）：<…>

复习完用 /swcc-stop 归档（掌握度由系统独立评估，自动记薄弱点并安排下次复习）。
⚠️ /swcc-go 和 /swcc-stop 需在同一轮对话里使用——stop 要读本次对话来评分。
```

---

## 质量基准（达到才算一次好复习）

- 覆盖清单里的核心概念基本都考到了（而非只问一两题）。
- 至少对关键概念做过「为什么/取舍/边界」的追问，不是停在背结论。
- 收尾小结结构清晰、如实标注覆盖与卡壳，stop 能据此客观打分。
