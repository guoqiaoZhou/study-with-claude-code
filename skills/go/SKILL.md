# swcc · go — 开始复习（先讲后测）

发起一次复习会话。**这是一次对话内的复习**:本技能围绕一个节点**先带你过一遍知识点、再考核**;复习结束后用 `/swcc-stop` 归档。

> 开始前先读数据契约:`${CLAUDE_PLUGIN_ROOT}/skills/_shared/data-contract.md`。本技能**只读不写**——所有进度/掌握度的落盘由 `/swcc-stop` 完成。

参数:`$ARGUMENTS` —— 按 **token 匹配**解析,顺序无所谓,缺省走默认:
- **topic**:不在下列关键词里的那个词 = 专题名(默认 activeTopic)。
- **取向**(学还是考):`study`=先讲后测(**默认**);`test`=直接考、跳过讲解。
- **题型**(只作用于考核阶段):`concept` / `scenario` / `mixed`(**默认**)。

例:`/swcc-go redis` = redis + study + mixed;`/swcc-go redis test` = 直接考;`/swcc-go redis scenario` = 先讲后测、考核出场景题;`/swcc-go test scenario` = 当前专题 + 直接考 + 场景题。

**三种切换取向的方式**:
1. **调用时传参**:如上 `/swcc-go redis test`。
2. **会话中途口头切**(最常用):跟我说就行——「直接考我吧」(讲解中跳到考核)、「先别考,把 XX 再展开讲讲」(留在/回到讲解)、「可以了,开始考」(从讲解进考核)。
3. **自动**:定位到的是**到期薄弱点**(已学过)→ 直接进考核(相当于 `test`);**生疏/首次**的节点 → 默认先讲后测(`study`)。

---

## 核心原则

1. **先学后测,别上来就考。** 默认先带用户把知识点过一遍,再考核。会背≠掌握,但**没过一遍就考会把"还没复习"冤枉记成一堆薄弱点**。
2. **讲解是 blog 级,不是聊天。** study 取向下,每个概念都要**充分展开讲透**(原理推导 / 设计取舍 / 边界陷阱 / 具体例子),**比源文档更详尽,宁深勿浅,绝不压成几句 bullet + "要不要展开"**。用户想深挖哪点就就地下钻、同样讲透。
3. **只有考核阶段算数。** 掌握度/薄弱点**只看考核阶段**的表现;讲解阶段不评判、不记薄弱点。
4. **覆盖该节点的关键概念。** 围绕一份核心概念清单逐个过、逐个考,别随机碰一两个就散。
5. **一次一个概念 / 一题。** 讲解一次推进一个概念,考核一次问一题。
6. **挂了书就用书。** 讲解与出题以原文为据;没书则靠自身理解,但不假称来源。
7. **为 stop 铺路。** 收尾小结要把「已讲解」与「考核结果」分开,薄弱点候选只来自考核阶段。

## 执行流程

| 阶段 | 名称 | 目的 |
|---|---|---|
| 1 | 加载专题 | 解析参数、读数据 |
| 2 | 定位节点 | 按优先级选当前该复习的节点 |
| 3 | 建覆盖清单 | 取该节点核心概念作为 checklist |
| 4 | **讲解（先过一遍）** | 逐概念讲透、可下钻;**不记薄弱点** |
| 5 | **考核（后测）** | 苏格拉底式追问验证;**只有这里算掌握度/薄弱点** |
| 6 | 收尾交接 | 结构化小结，提示 stop |

---

### 阶段 1：加载专题

- 先按上面「参数」对 `$ARGUMENTS` 做 token 解析,得到 topic / 取向(study|test) / 题型(concept|scenario|mixed)。
- topic 缺省 → 读 `$HOME/.study-with-cc/config.json` 的 `activeTopic`（config 不存在或缺 activeTopic 时，按数据契约第九节「兜底」扫 `topics/` 自愈，不要直接报错）。
- 该专题不存在 → 提示先 `/swcc-plan <topic>`，停止。
- 读 `progress.json`、`knowledge-tree.md`、`knowledge-system.md`、`references.json`。

### 阶段 2：定位当前复习节点

| 优先级 | 选谁 |
|---|---|
| 1 | `weakPoints` 中 `nextReview` ≤ 今天（`date +%F`）的薄弱点——优先复习到期薄弱点 |
| 2 | 否则 `currentNode` 指向的节点（若为 `in_progress`） |
| 3 | 否则下一个 `not_started` 的叶子节点 |

> 注:到期**薄弱点**已学过,定位到它时可直接进考核(相当于 `test` 取向);生疏/首次的节点走默认「先讲后测」。

### 阶段 3：建该节点的核心概念覆盖清单

从 knowledge-system.md 取**当前节点的「核心概念」清单**作为本次内部 checklist——讲解和考核都围绕它逐个推进(也是 stop「覆盖度」评分的依据)。挂了书则按 data-contract 第八节读相关章节页备用。

宣布定位信息:
```
🎯 当前复习：<topic> / <节点路径>
📅 上次复习：<lastReviewed 或「首次」>
🧭 取向：<study 先讲后测 / test 直接考>　核心概念：<x> 个
```

### 阶段 4：讲解（先过一遍，**blog 级深度**，study 取向；test 取向跳过本阶段）

带用户把核心概念清单**逐个过一遍**。**这不是聊天式寒暄,是 blog 级的讲解**。

⚠️ **knowledge-system.md 只是大纲/锚点(地图),不是教材**(见 data-contract 第四节)。讲解深度**必须超过 system 本身**:以它列的要点为骨架,**深度来自——挂载的书(读对应章节原文)+ 你自身的领域知识**,把每个要点展开讲透。只复述 system 的浓缩要点 = 不合格。

**逐个概念**,每个概念输出一段 blog 级讲解（宁深勿浅），包含:
- **是什么 + 为什么这样设计**:讲原理、给推导/来龙去脉,不止给结论。
- **设计取舍 / 对比**:和替代方案比,为什么是它。
- **边界、陷阱、易错点**:主动点出用户容易栽的细节。
- **具体例子**:命令 / 代码 / 真实场景;挂了书就结合原文(按 data-contract 第八节读相关章节页)并说清出处。
- 充分吸收 knowledge-system.md 该概念的内容**再加你自己的扩展**——产出要**比源文档更展开,不是更简略**。

讲完一个概念再**一句话 check-in**:邀请深挖具体子点(如「要不要把 GeoHash 编码 / listpack→hashtable 转换阈值再往下挖?」)。用户想深挖哪点 → **就地下钻、同样 blog 级讲透**,再继续下一个。

- **一次一个概念**(每个本身就可能很长,正常)。**不要把多个概念压在一条里草草带过**,也不要一口气把所有概念倒成一大篇——逐个、每个都讲到位。
- **本阶段不评判对错、不记薄弱点**,纯学习。
- 清单过完(或用户说「可以了,考我吧」)→ 进入阶段 5。`test` 取向或「直接考我」→ 跳过本阶段。

> 反例(别这样):把"五种基础+五种扩展类型"压成一个 5 行 bullet + 「要不要展开?」。
> 正解:逐个把每种扩展类型**寄生在谁之上、为什么、带来什么性能/扩容特征**讲清楚,该举例就举例,讲到能让用户接着往下深挖的程度。
> 提醒(原则 3):讲解再深也不算掌握——检验在下一阶段。

### 阶段 5：考核（后测，苏格拉底式，一次一题）

过完知识点后,围绕同一份清单逐个概念考核。**每个概念不是问一题就过**,按回答质量动态调难度:

| 用户回答 | 你的下一步 |
|---|---|
| 答得准且深（讲清原理/取舍/边界） | 认可 → 升一档:追问边界/反例/与替代方案对比，或换下一个概念 |
| 答得对但浅（只背了结论） | 追问「为什么会这样」「换个场景还成立吗」，逼出原理 |
| 卡壳 / 答错 | 先给小提示再追;仍卡 → 简要补讲，**心里标记为本次薄弱点候选** |

- 题型按 mode:`concept` 偏原理推导、`scenario` 给真实工程场景、`mixed` 交替。
- 每题:提问 → 等答 → **点评**（对/错/缺漏，必要时补讲取舍）→ 下一题。
- **只有本阶段的表现计入掌握度与薄弱点**(原则 3)。刚在阶段 4 讲过、这里一考就会的,说明真懂了;讲过还卡的,才是真薄弱点。

> 提醒（原则 1）:用户答「对」时别急着夸完就走,再问一层「为什么」往往见真章。

### 阶段 6：收尾交接

当节点关键概念已考核充分,或用户表示告一段落时,输出**结构化小结**（喂给 `/swcc-stop` 评估子智能体的关键输入）:

```
📝 本次复习小结（供 stop 评估用）
- 节点：<topic> / <节点路径>　取向：<study/test>　题型：<mode>
- 已讲解的概念：<逐个列>（仅过了一遍，不据此判薄弱）
- 考核结果：<逐个概念标注 ✅扎实 / ⚠️答浅 / ❌卡壳——薄弱点只从这里取>
- 答得好的点：<…>
- 讲过仍卡的点（候选薄弱点）：<…>

复习完用 /swcc-stop 归档（掌握度由系统独立评估，自动记薄弱点并安排下次复习）。
⚠️ /swcc-go 和 /swcc-stop 需在同一轮对话里使用——stop 要读本次对话来评分。
```

---

## 质量基准（达到才算一次好复习）

- 先把核心概念**过了一遍**(study 取向),且讲到了易忽略的细节,而不是上来就考。
- 过完之后**考核**了同一批概念,薄弱点只来自考核阶段(不会把"还没复习"误记为薄弱)。
- 收尾小结把「已讲解」与「考核结果」分开,stop 能据此客观评分、只对考核结果判薄弱。
