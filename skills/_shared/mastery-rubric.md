# 掌握度评分准则（Mastery Rubric）

> 本文件供 `stop` 技能在「独立掌握度评估」阶段使用。`stop` 派发一个**独立评估子智能体**，把本轮复习对话的事实（含 `go` 的收尾小结）连同本准则交给它，由它客观打分并返回结构化结果。**掌握度完全由系统评估，绝不让用户自评。**

---

## 一、为什么用独立 subagent 评估

主 Claude 是本轮复习的「考官」，对用户的回答**容易心软打高分**（刚讲解过、印象新鲜、想鼓励）。换一个**没参与对话、只看事实记录**的子智能体打分，更接近客观。这是技能用流程保证评分公正，不是让用户自评。

---

## 二、两个评分维度（各自 1–10，再合成 mastery）

### 维度 A：知识面覆盖度（coverage）
本次实际触及该节点的**关键概念占比**——参照该节点 knowledge-system.md 的「核心概念」清单与 `go` 收尾小结里「覆盖了哪些概念」。

| 分段 | 含义 |
|---|---|
| 1–3 | 只碰到该节点一两个点，大片关键概念没涉及 |
| 4–6 | 覆盖了一半左右的关键概念 |
| 7–8 | 覆盖了大部分关键概念，少量遗漏 |
| 9–10 | 关键概念基本全覆盖 |

### 维度 B：回答深度（depth）
用户回答停留在哪一层——背诵结论 / 讲清原理 / 能谈取舍与边界 / 能结合场景。

| 分段 | 含义 |
|---|---|
| 1–3 | 答不上来，或只蹦名词、背结论 |
| 4–6 | 能复述原理，但讲不清「为什么」「取舍」「边界」 |
| 7–8 | 讲清了原理与设计取舍，能举例 |
| 9–10 | 原理 + 取舍 + 边界 + 真实场景都清楚，能反驳/比较替代方案 |

---

## 三、合成 mastery（1–10）

- 基线:`mastery = round(coverage × 0.4 + depth × 0.6)`（深度权重更高——会背不等于掌握）。
- 重要约束:**任一维度 ≤ 3 时，mastery 不得高于 5**（覆盖太窄或答得太浅都不能算掌握）。
- `mastery ≥ 8` 才视为 `mastered`（且需满足状态机:连续两次达标才从薄弱点移除，见 data-contract）。

---

## 四、薄弱点判定

> **只看考核阶段。** go 默认先讲后测——仅在「讲解/复习过、考核时仍卡壳或答浅」时才记薄弱点。光过了一遍讲解、还没考核的概念**不记薄弱点**(用户只是刚复习,不代表不会)。

把以下点写入/保留为 `weakPoints`:
- 考核时**卡壳、答错、或明显答浅**的具体概念；
- 该节点 mastery `< 6` 时，把考核中暴露的薄弱概念都记上。

每个薄弱点带 `node / concept / mastery / createdAt / nextReview / reviewCount / consecutivePass`（见 data-contract 第五节）。**子智能体只产出候选(`node/concept/mastery`)**,其余字段由 stop 主体补齐/继承。

---

## 五、评估子智能体的输入与输出

**输入**（由 stop 主体整理后交给子智能体）:
- 专题、节点路径、本次 mode；
- 该节点的核心概念清单（来自 knowledge-system.md）；
- 本轮对话里:问了哪些题、用户怎么答、哪里卡壳/答错/答好（含 `go` 收尾小结）。

**输出**（子智能体返回的 JSON）:

```json
{
  "mastery": 7,
  "coverage": 6,
  "depth": 8,
  "reasons": "考核覆盖了该节点约一半的核心概念；其中两处能讲清原理与取舍并举出例子，故深度评分偏高；但有一个关键机制完全没被问到，覆盖度因此中等。",
  "weakPoints": [
    { "node": "<该薄弱概念所属节点路径>", "concept": "<考核时卡壳或只答出结论的那个具体概念>", "mastery": 4 }
  ]
}
```

> 子智能体只需返回 `node / concept / mastery`(薄弱点候选);`createdAt / nextReview / reviewCount / consecutivePass` 等字段由 **stop 主体补齐**(新点初始化、已存在的点继承旧计数,见 stop 阶段 4)。stop 主体拿到结果后，负责写 review-session（含 reasons）、更新 progress.json（mastery/status/nextReview/weakPoints/stats）、按需勾选 knowledge-tree.md。子智能体只评估、不写文件。
